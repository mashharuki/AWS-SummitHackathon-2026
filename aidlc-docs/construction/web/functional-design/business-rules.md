# ビジネスルール — U-05: web

**Unit**: U-05: web
**ステージ**: CONSTRUCTION / Functional Design
**作成日**: 2026-05-17
**バージョン**: 1.0.0

---

## 1. 認証ルール

### BR-WEB-01: 未認証リダイレクト
- 認証済みセッション（Cognito JWT）がない状態でアプリにアクセスした場合、即座に `/login` へリダイレクトする
- OAuth コールバック URL（`/auth/callback`）は認証ガードを免除する

### BR-WEB-02: トークン自動リフレッシュ
- アクセストークンの有効期限切れ時は自動的にリフレッシュトークンで更新する
- リフレッシュトークンも失効した場合はログアウト処理を実行し、ログイン画面へリダイレクトする

### BR-WEB-03: Google OAuth フロー
- Cognito Hosted UI へのリダイレクト前に `state` パラメータを生成・localStorage に保存する（CSRF対策）
- コールバック時に `state` を検証してから Authorization Code を Cognito に送信する

## 2. タスク一覧ルール

### BR-WEB-04: タスクセクション分類
- APIレスポンス（`GET /api/tasks?status=pending`）で取得した候補は「承認待ちタスク」セクションに表示する
- APIレスポンス（`GET /api/tasks?status=approved`）で取得したタスクは「承認済みタスク」セクションに表示する

### BR-WEB-05: 楽観的更新（Optimistic Update）
- 承認・削除操作はAPIレスポンスを待たずにUI上即座に反映する（楽観的更新）
- API失敗時はロールバックしてトースト通知で失敗を伝える

### BR-WEB-06: インライン編集
- 「編集」ボタン押下で対象カードがインライン編集モードに切り替わる
- 他のカードが編集中の場合、そのカードを先にキャンセルしてから新しい編集を開始する
- 「保存して承認」は PATCH + Approve を1トランザクションで実行する（2エンドポイント連続呼び出し）

### BR-WEB-07: バナーメッセージ
- 当日の承認済みタスクのうち `verdict: 'can_saboru'` が最多であれば「今日はサボれます！」バナーを表示する
- `danger` が1件以上あれば「今日は要注意」バナーに切り替える

## 3. タスク詳細ルール

### BR-WEB-08: サボり提案の自動取得
- タスク詳細画面を開いた際に `GET /api/tasks/:id/proposal?stream=true` を自動呼び出しする
- SSEストリーミングでテキストをリアルタイムに右ペインのチャット欄に表示する
- ストリーミング中はチャット入力欄をdisabledにする

### BR-WEB-09: チャット初期メッセージ
- タスク詳細を初めて開いた時、「このタスクは安全に後日できるよ。安心してサボれます。」のようなサボローのウェルカムメッセージをチャット最上部に表示する

### BR-WEB-10: 判定色分けルール
- `verdict: 'can_saboru'` → 緑系（`bg-green-50` / バッジ `bg-green-500`）
- `verdict: 'caution'` → 黄色系（`bg-yellow-50` / バッジ `bg-yellow-500`）
- `verdict: 'danger'` → 赤系（`bg-red-50` / バッジ `bg-red-500`）

### BR-WEB-11: クイック返信後のフロー
- クイック返信ボタン押下 → `POST /api/tasks/:id/honne` → サボローの返答をチャット欄に追加
- 返答の下部に「この反応が積み重なって、将来あなただけの取扱説明書になります」テキストを表示する

### BR-WEB-12: 自由テキスト送信バリデーション
- 空文字の送信は禁止。送信ボタンをdisabledにする
- 500文字以上の入力はカウンター表示と送信禁止でユーザーに通知する

## 4. 設定画面ルール

### BR-WEB-13: 連携ステータス表示
- `ServiceConnection.status === 'active'` → 「連携済み」緑バッジ表示
- `ServiceConnection.status === 'expired'` → 「再連携が必要です」オレンジバナー表示
- 未連携 → 「連携する」ボタン表示

### BR-WEB-14: Slack OAuth フロー
- 「Slack を連携する」クリック → `GET /api/connections/slack/auth-url` で認証URL取得 → リダイレクト
- コールバック後 `GET /api/connections` で連携状態を再取得して表示更新する

### BR-WEB-15: ログアウト
- 「ログアウト」クリック → Cognito signOut → localStorage/sessionStorage クリア → `/login` リダイレクト

## 5. グローバルルール

### BR-WEB-16: エラーハンドリング統一
- 全APIエラー（4xx / 5xx）はトースト通知で1件表示する（スタック禁止）
- ネットワークエラーは「接続できませんでした。再試行してください」メッセージを表示する
- 401エラーはトークンリフレッシュを試み、失敗時はログアウト処理を実行する

### BR-WEB-17: トースト通知
- 成功: 3秒後自動消滅
- エラー: 5秒後自動消滅（手動クローズも可）
- 同時表示は最大1件（新しいトーストが既存を置き換える）

### BR-WEB-18: Three.js 演出表示条件
- Three.js キャラクターはタスク詳細右ペインのサボローアイコン位置に表示する
- SSEストリーミング中はアニメーションを活性化（`isStreaming: true`）
- `verdict` が確定したらそれに応じたアニメーションパターンへ切り替える
- 低性能デバイス（`prefers-reduced-motion: reduce`）では静止画フォールバックを使用する

### BR-WEB-19: レスポンシブ対応
- タスク詳細は 768px以上でサイドバイサイド（左右2ペイン）、768px未満ではタブ切替（詳細タブ / チャットタブ）
- タスク一覧はモバイルファースト（単一カラム）
