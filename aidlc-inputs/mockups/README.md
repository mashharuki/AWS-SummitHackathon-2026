## 画面1: タスク一覧

### ヘッダー
- ロゴ：🛏️ アイコン + 「**SABOROU**」（プロダクト名・英字大文字）
- タグライン：「- サボりの最適解 -」

### コンテンツ
- **「承認済みタスク」セクション**
  - タスクカード形式（締切・1行のサボり判定サマリ）
- **「承認待ちタスク」セクション**
  - タスクカード形式（締切・タスク説明）
  - **アクションボタン3種**：「承認する」（オレンジ・主CTA）/「編集」（白・セカンダリ）/「削除」（赤系・破壊的）

### フローティングボタン
- 右下に **+ ボタン（FAB）**（オレンジ）：手動でタスクを追加可能

### 配色
- 背景：オフホワイト（#FAFAFA系）
- 主CTA：オレンジ（#F97316 系）
- 削除：赤
- カード：白＋淡いシャドウ
- テキスト：濃いグレー

## 画面2: タスク詳細・チャット（左右2ペイン）

### ヘッダー
- 同じく SABOROU + タグライン

### 左ペイン：タスク詳細
- 「< 一覧に戻る」リンク
- タスク名（大見出し）
- **「タスクの前提」ボックス**
  - 締切（具体的な時刻と注意事項）
  - やること（成果物の明確な定義）
  - 誰が言っているか（依頼者・出典・タイムスタンプ）
- **「現在のサボり判定」**（淡黄色の強調ボックス）
  - 1行で簡潔な判定文
- **「判断材料」**
  - 箇条書きの根拠（複数項目）

### 右ペイン：チャット
- 「チャット」見出し（左）と「SABOROU と相談」リンク（右）
- **SABOROU からのメッセージ**（オレンジ枠の名前ラベル + 本文）
- 下部に固定された応答エリア：
  - **クイック返信ボタン4種**：「たしかに、まだ寝かせる」/「いや、このタスクは早めにやった方がいい」/「15分だけやる」/「完全に放置したい」
  - **自由入力欄** + 「送信」ボタン（オレンジ）

## 重要なUXポイント

1. **「サボり判定」が一覧にも詳細にも表示される**
   - 一覧：1行サマリ（軽量・即時表示）
   - 詳細：判定 + 判断材料の箇条書き（深い説明）

2. **「判断材料」が SABOROU の説得力を作る**
   - 「なぜ今サボれるか」を具体的に示すことで、企画書 §4 の本質と整合

3. **承認 / 編集 / 削除の3アクション**
   - タスク候補に対するユーザーの主体性を担保

4. **クイック返信 + 自由入力の二段構え**
   - 速い反応も詳細な反応も同じ画面で受け取れる
   - 本音データの収集チャネルとして機能

5. **デスクトップ前提のレイアウト**
   - 左右2ペインはモバイルでは縦積みに切り替えが必要
   - レスポンシブ要件は Application Design で確定

## モック上の例示データ（参考）

- **タスク例1**: 「提案資料の初稿作成」（締切：明日 12:00、依頼者：営業の田中さん、Slack DM 昨日 17:42）
- **タスク例2**: 「議事録の共有」（締切：今日 18:00、午前中の定例MTG関連）
- **SABOROU の発話例**: 「これは完全放置すると危ないけど、今日中に全部やる必要はなさそうだよぉ。今は構成だけ作って寝かせるのがよさそう。」
- **判断材料の例**:
  - まだリマインドは来ていない
  - 「至急」のような強い文言は確認されていない
  - 関連会議は明日午後
  - 他メンバーの進捗もまだ未確定
